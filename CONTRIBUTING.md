# Contribuir

Gracias por tu interés en este proyecto.  
Este repositorio prioriza la claridad y la estabilidad.

- Abre una *issue* antes de proponer cambios grandes.  
- Las *pull requests* deben incluir una descripción clara y pruebas básicas.  
- No se aceptan binarios ni ejecutables en PRs; los releases se gestionan por el autor.  
- Mantén el código y la documentación concisos y pedagógicos.
